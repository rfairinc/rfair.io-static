Deploying Static Websites with Github Pages and WP2Static

December 15, 2019 by Russell

Recently, I decided to give WP2Static a try. Results: I love it. In fact, I am using it to deploy a static version of this website to Github Pages.

Why static sites? Four main reasons.

First, Security. Because the site pages are static html, and don’t “interact” with a traditional server there are far fewer ways for a potential attacker to compromise the website.

Second, Performance. The way WP2Static works, is that it generates the html for each page on the website once, then saves the rendered html. This makes page loading much faster, and the rendered html can be distributed to edge nodes on a CDN.

Third, Reliability. Because there aren’t any typical servers running, there isn’t much that can go down. Sure if something happens at the dns level, or Github goes down, the website would also go down, but that’s the case in a non-static environment too. By putting a CDN like Cloudflare in front of the static content, this risk is mostly mitigated.

Fourth, Cost. Github Pages is free, and can scale well when service static html.
